# 满二叉树
满二叉树就是每一层节点都是慢的，像个正三角形
![[Pasted image 20260401143200.png]]
**满二叉树有个优势，就是它的节点个数很好算**。假设深度为 `h`，那么总节点数就是 `2^h - 1`，等比数列求和嘛，我们应该都学过的。

# 完全二叉树
完全二叉树指的是: 二叉树的每一层节点<span style="background:#ff4d4f">都紧凑靠左排列</span>，且除了最后一层，其他每一层都必须是满的
![[Pasted image 20260401143338.png]]
况且你还可以发现，其实满二叉树就是一种特殊的完全二叉树
完全二叉树有一个特点就是：由于它节点紧凑排列，如果对它进行从上到下，从左到右的每一层的每一个节点进行排序，那么父子节点可以通过一个式子表示，并且可以使用数组进行存储，具体式子为:
1. 若节点编号从 0 开始：
	- 父节点到左子节点：`leftChild(i)=2i`
	- 父节点到右子节点：`rightChild(i)=2i+1`
2. 若节点编号从 1 开始：
	- 父节点到左子节点：`leftChild(i)=2i+1`
	- 父节点到右子节点：`rightChild(i)=2i+2`
然后还有一个比较难发觉的满二叉树的特性就是，<span style="background:#ff4d4f">左右子树，至少有一棵树是满二叉树</span>

# 二叉搜索树(Binary Search Tree)

对于树中的每个节点，其<span style="background:#ff4d4f">左子树的每个节点</span>的值都要小于这个节点的值，<span style="background:#ff4d4f">右子树的每个节点</span>的值都要大于这个节点的值。你可以简单记为「左小右大」。

# 高度平衡二叉树
高度平衡二叉树（Height-Balanced Binary Tree）是一种特殊的二叉树，**它的「每个节点」的左右子树的高度差不超过 1**。
> <span style="background:#ff4d4f">要注意是每个节点</span>，而不仅仅是根节点。

# 自平衡二叉树
当通过一些增删改的操作，二叉树经历旋转或者改色的操作，使得二叉树的高度始终保持高度差为 1 的树，经典的有红黑树，和 AVL 树

# 二叉树的递归/层序遍历

二叉树只有这两个遍历，递归遍历可以衍生出 DFS 算法，层序遍历可以衍生出 BFS 算法

## 递归遍历
> 递归遍历的顺序是固定的，但是有三个关键位置，在不同的位置插入代码，会产生不同的效果

```
// 基本的二叉树节点
class TreeNode {
public:
    int val;
    TreeNode* left;
    TreeNode* right;

    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};

// 二叉树的递归遍历框架
void traverse(TreeNode* root) {
    if (root == nullptr) {
        return;
    }
    traverse(root->left);
    //你会发现，由于代码顺序问题，导致了这个函数始终是先遍历完左子树，再去遍历右子树
    //你当然可以去调换这两行代码的顺序，这都随你
    traverse(root->right);
}
```
但是递归遍历有前/中/后遍历

```
// 基本的二叉树节点
class TreeNode {
public:
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};

// 二叉树的递归遍历框架
void traverse(TreeNode* root) {
    if (root == nullptr) {
        return;
    }
    //前序位置
    traverse(root->left);
    //中序位置
    traverse(root->right);
    //后序位置
}
```
<span style="background:#ff4d4f">前序位置的代码会在进入节点时立即执行；中序位置的代码会在左子树遍历完成后，遍历右子树之前执行；后序位置的代码会在左右子树遍历完成后执行</span>:
![[Pasted image 20260401150534.png]]
>  实际的算法题中不会简单的让你计算前中后序的遍历结果，而是需要你把正确的代码写到正确的位置，<span style="background:#d3f8b6">所以你必须准确理解三个位置的代码产生的不同效果</span>，才能写出准确的代码。

## 层序遍历
上面讲的递归遍历是依赖函数堆栈递归遍历二叉树的，遍历顺序是从最左侧开始，一列一列地走到最右侧。
二叉树的层序遍历，顾名思义，就是一层一层地遍历二叉树：![[Pasted image 20260401150823.png]]
## 简单遍历
```
class TreeNode {
public:
  int val;
  TreeNode *left;
  TreeNode *right;
  TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};
void levelOrderTraverse(TreeNode *root) {
  if (root == nullptr) {return;}
  std::queue<TreeNode *> q;
  q.push(root);
  while (!q.empty()) {
    TreeNode *cur = q.front();
    q.pop();
    std::cout << cur->val << std::endl;
    if (cur->left != nullptr) {
      q.push(root->left);
    }
    if (cur->right != nullptr) {
      q.push(root->right);
    }
  }
}
```
> 这种写法的最大的优点就是简单，每次把队列的头元素拿出来，再把左右子节点加入队列即可
> 但是这种写法最大的问题就是<span style="background:#ff4d4f">不知道这棵树的深度</span>，当题目需要这棵树的最小深度或者是收集某一层的节点时，会出问题，

## 带有深度的层序遍历
```
void levelOrderTraverse(TreeNode *root) {
  if (root == nullptr) {
    return；
  }
  std::queue<TreeNode *> q;
  int depth = 1;
  while (!q.empty()) {
    int sz = q.size();
    
    for (int i = 0; i < sz; i++) {
      TreeNode *cur = q.front();
      q.pop();
      std::cout << "depth = " << depth << ", val = " << cur->val << std::endl;
      if (cur->left != nullptr) {
        q.push(root->left);
      }
      if (cur->right != nullptr) {
        q.push(root->right);
      }
    }
    depth++;
  }
}
```
这种可以适配多种题目，比如二叉树最小深度这种问题，但是需要注意的是，你一定要先把队列的长度保存下来，因为整个 `for` 循环中队列的长度是会变化的

## 树枝深度层序遍历
```
class State {
public:
  TreeNode *node;
  int depth;
  State(TreeNode *node, int x) : depth(x), node(node) {}
};
void levelOrderTraverse(TreeNode *root) {
  if (root == nullptr) {
    return;
  }
  std::queue<State> q;
  q.push(State(root, 1));
  while (!q.empty()) {
    State cur = q.front();
    q.pop();
    std::cout << "depth = " << cur.depth << ", val = " << cur.node->val<< std::endl;
    
    if (cur.node->left != nullptr) {
      q.push(State(root->left, cur.depth + 1));
    }
    if (cur.node->right != nullptr) {
      q.push(State(root->right, cur.depth + 1));
    }
  }
}
```

<span style="background:#affad1">回顾写法二，我们每向下遍历一层，就给 `depth` 加 1，可以理解为每条树枝的权重是 1，二叉树中每个节点的深度，其实就是从根节点到这个节点的路径权重和，且同一层的所有节点，路径权重和都是相同的。</span>
那么假设，如果每条树枝的权重可以是任意值，现在让你层序遍历整棵树，打印每个节点的路径权重和，你会怎么做？
这样的话，同一层节点的路径权重和就不一定相同了，写法二这样只维护一个 `depth` 变量就无法满足需求了。
写法三就是为了解决这个问题，在写法一的基础上添加一个 `State` 类，让每个节点自己负责维护自己的路径权重和。
